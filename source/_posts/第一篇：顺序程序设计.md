# 一、一切的开始，"Hello World!", 你好世界！

先看下面这段最简单的代码，学一门编程语言我们一般都会从输出 `Hello World!` 开始，现在用你的 `Dev_C++` 按照下面的代码，一步一步的键入你的代码，并成功运行，在你的终端里看到打印出来的`Hello World!`

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Hello World!" << endl;
    return 0;
}
```

### 1、 `#include <iostream>` 这个地方在做什么呢？

`#include <iostream>` 这个地方在做什么呢？

`#include` 是告诉电脑：“我要用别人已经写好的工具”。这个工具的名字叫 `iostream`，用尖括号 `< >` 包起来，意思是“去系统目录里找”。

`iostream` 的全称是 **input output stream**，输入输出流。它里面装着 C++ 里最基本的输入输出工具——你马上要用的 `cout` 和 `cin` 都在里面。

> 一个不太严谨但好懂的理解：`#include <iostream>` 就像是你在考试前找学长借了一本“万能笔记”，里面记好了各种常用操作的写法，你直接翻着用就行。

------

### 2、`using namespace std;` 这行是干嘛的？

这行看起来有点抽象，但它的作用和意思却很简单：**为了省事儿**。

C++ 标准库（就是C++官方给大家写好的“万能笔记”）里的所有东西都放在一个叫 `std`（standard 的缩写）的“命名空间”里。如果不写这行，你每次用 `cout` 都得写成 `std::cout`，每次用 `cin` 都得写成 `std::cin`。

有了 `using namespace std;`，你就可以直接写 `cout` 和 `cin`，不用每次都加 `std::` 前缀。

> 你可以把 `std` 想象成一个很大的工具箱，`using namespace std;` 就是你把工具箱打开放在脚边，里面的工具伸手就能拿到，不用每次都喊全名。

------

### 3、`int main()` 是什么？

这是**程序的入口**。

C++ 程序启动时，电脑会去找一个叫 `main` 的函数，从它开始执行。没有 `main`，程序就不知道从哪开始跑。

- `int` 表示这个函数执行完会返回一个整数（所以`return 0`就是告诉操作系统“我运行完了”）
- `()` 表示这是一个函数，括号里可以放参数（这里暂时为空）
- `{ }` 里面包着的，就是程序真正执行的代码

你写的所有代码，最终都放在 `main` 函数的这对大括号里。

------

### 4、`cout << "Hello World!" << endl;` 这行在做什么？

`cout` 是 C++ 里的**标准输出流**，可以理解成“输出到屏幕”。

`<<` 是**插入运算符**，意思就是“把后面的东西送到 `cout` 里”。

`"Hello World!"` 是要打印的字符串。

`endl` 是换行，相当于按了一下回车。

整句话连起来就是：**把 `Hello World!` 打印到屏幕上，然后换行**。

> 注意结尾的 `;` —— C++ 里每一条语句结束都要写分号，漏了就会编译报错。

------

### 5、`return 0;` 是做什么的？

`return 0` 表示 `main` 函数执行完毕，返回 `0` 给操作系统。

在 C++ 里，`0` 通常表示“程序正常结束”。如果返回其他数字，表示程序出了某种错误。

这行不是必须的——编译器会自动帮你补一个隐式的 `return 0`。但**建议写上**，这是一个好习惯，也是很多 OJ 平台的要求。

------

### 把整段代码串起来

现在回头看整段代码，你应该能读懂每一行的作用了：

```cpp
#include <iostream>   // 借用 iostream 里的工具
using namespace std;  // 省掉 std:: 前缀

int main() {          // 程序从这里开始执行
    cout << "Hello World!" << endl;  // 打印并换行
    return 0;         // 告诉系统：我正常结束了
}
```

这就是一个完整的 C++ 程序。它做的事情很简单——在屏幕上打印一行字。

但在这背后，你已经接触了 C++ 里的三个核心概念：

- **头文件**（怎么借用别人的代码）
- **主函数**（程序从哪里开始）
- **输入输出流**（怎么和外界说话）

这三个东西，会一直跟着你学完整个 C++。

# 二、从 A + B 认识数据类型

`Hello World` 只是让程序“说话”，但程序真正的价值在于**处理数据**。而处理数据的第一步，就是理解数据在程序里是如何被存放和操作的。这一节，我们通过一个经典的问题来入门：**A + B problem**。

> 输入两个整数，输出它们的和。
>
> 这是几乎所有 OJ（Online Judge，在线判题系统）的第一道题，也是你入门的第一个台阶。

**先看代码**



```cpp
#include <iostream>
using namespace std;

int main() {
    int a, b;          // 声明两个整数变量
    cin >> a >> b;     // 从键盘读入两个数，分别存入 a 和 b
    int c = a + b;     // 计算 a + b 的结果，存入 c
    cout << c << endl; // 输出结果
    return 0;
}
```

这段代码做的事情，和它的逻辑一样直白：

1. 准备两个“盒子`a` 和 `b`，用来装整数
2. 从键盘读入两个数，分别放进 `a` 和 `b`
3. 计算 `a + b`，把结果放进 `c`
4. 输出 `c`

### 1、什么是变量

变量可以理解成一个**带名字的盒子**：

- 盒子上贴着名字，叫 `a`
- 盒子里存着东西，比如 `5`
- 你可以随时往盒子里换新东西

在 C++ 里，变量必须先声明再使用：

cpp

```
int a;          // 声明一个名叫 a 的盒子，用来装整数
a = 5;          // 往盒子里放 5
int b = 3;      // 声明盒子 b，同时放 3（这叫“初始化”）
```



**声明和赋值是两件事**，新同学容易混淆：

cpp

```
int a;      // 声明 —— 告诉编译器“我要一个盒子”
a = 5;      // 赋值 —— 往盒子里放东西
int b = 3;  // 初始化 —— 声明 + 赋值一步到位
```



------

### 2、数据类型：盒子的“大小”决定能装什么

在 C++ 里，不同的数据需要不同类型的变量来存储。你不能用一个装小东西的盒子去装一个大东西——内存里也一样。

最常用的几种数据类型：

| 类型        | 中文名       | 大小   | 范围（约）                    |
| :---------- | :----------- | :----- | :---------------------------- |
| `int`       | 整数         | 4 字节 | -2.1×10⁹ ~ 2.1×10⁹            |
| `long long` | 长整数       | 8 字节 | -9.2×10¹⁸ ~ 9.2×10¹⁸          |
| `double`    | 双精度浮点数 | 8 字节 | 带小数，15-16 位有效数字      |
| `char`      | 字符         | 1 字节 | 单个字母/符号，用单引号包起来 |
| `bool`      | 布尔值       | 1 字节 | `true` 或 `false`             |

#### 无符号整数：真正的“环形”

比如 `unsigned char` 的范围是 `0 ~ 255`。当数值超出这个范围时，它就像钟表的指针一样，会**自动回绕**：

cpp

```
unsigned char a = 255;
a = a + 1;        // 结果是 0
a = a - 1;        // 结果是 255
```



就像钟表从 `23:59` 走到 `00:00` 一样，`255 + 1` 会变成 `0`，`0 - 1` 会变回 `255`。这种“从最大值加一回到最小值”的特性，确实是名副其实的**环形**。

#### 有符号整数：更像是“溢出即陷阱”

而有符号整数（`int`、`long long`）的溢出行为在语言标准里被定义为**未定义行为**，也就是说编译器可以任意处理，不能依赖任何结果。

在绝大多数现代计算机上（采用补码表示法），`int` 的溢出表现确实也像环形：

cpp

```
int a = 2147483647;   // int 最大值
a = a + 1;            // 结果是 -2147483648（绝大多数情况下）
```



从 `+2147483647` 跨一步变成 `-2147483648`，看起来像是从环形的一端跳到了另一端。但**语言标准并不保证这个结果**，不同编译器或优化选项下可能有不同表现。

```
int a = 2147483647;        // int 能装的最大正数
int b = a + 1;             // 溢出！结果会变成负数
long long c = 2147483648;  // 用 long long 就没事
```

> 所以在 ACM 里，有符号整数的溢出被视为“未定义行为”，从工程角度来说，它更像一个**暗坑**，而不是可预测的环形。

> **无符号整数**的溢出是环形的——最大值加一会回到最小值，最小值减一会回到最大值。
>
> **有符号整数**（`int`、`long long`）理论上溢出的结果是未定义的，但现代计算机上通常表现为补码环绕。**不要依赖这种行为**，在写代码时应该主动避免溢出。

在 ACM 里，数据范围往往是决定用 `int` 还是 `long long` 的关键。赛场上因为 `int` 溢出而 WA（Wrong Answer），是每个选手都踩过的坑。

> **一个简单经验**：看到题目里数据范围超过 10⁹，直接用 `long long`，不要犹豫。

------

### 3、输入与输出

`cin` 和 `cout` 是 `iostream` 里提供的两个最基本工具：

- `cin >>`：从键盘读入数据（箭头指向变量，表示数据“流进”变量）
- `cout <<`：向屏幕输出数据（箭头指向 `cout`，表示数据“流出”到屏幕）

`cin` 会自动跳过空格和换行，所以输入 `3 4` 或：

```
3
4
```

都能正确读入。

------

### 4、浮点数与 `double`

如果题目涉及小数，用 `double`：

```cpp
double pi = 3.14159;
double r;
cin >> r;
cout << r * r * pi << endl;
```



> **注意**：ACM 里浮点数判断相等时，不要用 `==`，而是判断两个数的差的绝对值是否小于一个很小的数（如 `1e-9`）。浮点运算有误差，直接比较会出问题。
